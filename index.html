Projects and Reflection

01
PowerShell Microsoft 365 Enterprise Automation

Office 365 Deployment Automation

A modular PowerShell solution designed to download, install, and validate a standardized set of Microsoft 365 applications across Windows endpoints with minimal manual effort.

Project Goal

Create a repeatable deployment process that reduces inconsistent manual installation steps.

Tools Used

PowerShell, Windows administration, Microsoft 365 installers, command-line utilities, and logging.

Skills Learned

Defensive scripting, validation logic, reusable functions, silent installation, and technical documentation.

Outcome

Produced a structured workflow that can install approved applications and confirm whether deployment succeeded.

Reflection on the Learning Experience

This project taught me that automation must account for failure, not only the ideal path. I improved the solution by thinking about permissions, missing files, partial installations, and clear feedback for the technician running the script. I would next expand the reporting and add more automated tests for different endpoint conditions.

02
API Integration JSON Asset Management

Lenovo Warranty Lookup

A PowerShell tool that accepts device serial numbers, queries Lenovo warranty data, transforms the returned information, and prepares structured records for review.

Project Goal

Make warranty information easier to retrieve and organize during device and asset reviews.

Tools Used

PowerShell, REST API requests, JSON parsing, serial-number input, CSV export, and error handling.

Skills Learned

API authentication concepts, response validation, data transformation, structured export, and input checking.

Outcome

Created a reusable lookup process that turns raw warranty responses into information that is easier to compare.

Reflection on the Learning Experience

This project strengthened my understanding of how external services can return incomplete or unexpected data. I learned to validate both the request and the API response before exporting results. In a future version, I would add a more polished interface and clearer batch-processing progress for large device lists.

03
ITSM Workflow Design Service Operations

OpsFlow

A service-desk workflow concept that organizes incident queues, clarifies priority levels, and supports more consistent ticket ownership, escalation, and handoff practices.

Project Goal

Improve visibility into service-desk work and make ticket prioritization and handoffs easier to follow.

Tools Used

ITSM process mapping, Jira-style workflow concepts, queue organization, documentation, and interface planning.

Skills Learned

Requirements analysis, workflow modelling, operational prioritization, user-centred design, and process communication.

Outcome

Developed a clearer operating model for viewing queue status, responsibility, urgency, and next actions.

Reflection on the Learning Experience

OpsFlow showed me that process design is as important as technical implementation. A workflow can fail when categories are unclear or when users do not understand what happens next. I learned to simplify labels, make ownership visible, and design around how technicians actually make decisions during busy support periods.

04
Hack the North Augmented Reality Generative AI

SpectacularAR — Hack the North

A voice-driven augmented-reality music visualizer created with Lens Studio, the Gemini API, Snap 3D Object Gen, and physical Snap Spectacles hardware.

Project Goal

Transform a user’s spoken music idea into an interactive visual experience viewed through AR hardware.

Tools Used

Lens Studio, Gemini API, Snap 3D Object Gen, JavaScript, prompt design, and Snap Spectacles.

Skills Learned

Rapid prototyping, AR interaction design, API integration, hardware testing, team collaboration, and pitching.

Outcome

Delivered a working hackathon prototype that connected voice input, generative tools, and a wearable AR experience.

Reflection on the Learning Experience

Building SpectacularAR under hackathon time pressure taught me to prioritize the core user experience and test risky integrations early. I learned that a convincing prototype does not need every planned feature, but it must clearly demonstrate the central idea. I would next improve response speed, visual variety, and onboarding.
